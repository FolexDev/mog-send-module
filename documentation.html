1️⃣ Vue d'Ensemble du Système

MOG Send est une application web qui permet d'envoyer des PDFs individualisés par email à plusieurs personnes. Le système automatise entièrement le processus :

🎯 Objectif Principal
Permettre à un utilisateur de :
1. Sélectionner des PDFs dans un dossier Google Drive
2. Créer une liste de bénéficiaires (avec prénoms, noms, emails)
3. Attribuer automatiquement à chaque personne le(s) PDF correspondant(s)
4. Envoyer les emails avec les PDFs attachés via Make

🔑 Points Clés

  • Frontend : Application web (HTML/CSS/JavaScript) pour l'interface utilisateur
  • Backend : Scénarios Make pour orchestrer l'envoi des emails
  • Stockage : Google Sheets pour centraliser les données
  • Authentification : Google OAuth 2.0 pour accéder à Drive et Sheets
  • Deux Modes : Normal (1 PDF/personne) et 1by1 (plusieurs PDFs/personne)

3️⃣ Les Deux Modes de Fonctionnement

Mode 1-PDF (1 PDF par personne)

MODE NORMAL
Cas d'Usage
Chaque bénéficiaire reçoit un seul PDF avec un numéro attribué. Exemple : Gagnant 1 reçoit PDF n°1, Gagnant 2 reçoit PDF n°2, etc.
Exemple
Vous avez 100 PDFs numérotés : ticket-1 à ticket-100
Vous sélectionnez 10 gagnants
Gagnant 1 → ticket-1
Gagnant 2 → ticket-2
... Gagnant 10 → ticket-10
Configuration Valeur
Google Sheet 11O24DWCA2pWtEh344R1ABJ10ZsArKCEiBNWC5EWV2wY
Scénario Make OA Envoi 1-PDF
Cellule Progrès 'Liste envois'!E2 (Format: "X/Y")
Colonnes Gagnants Numéro | Prénom | Nom | Email

Mode N-PDF (Plusieurs PDFs par personne)

MODE 1BY1
Cas d'Usage
Chaque bénéficiaire reçoit un nombre variable de PDFs défini manuellement. Exemple : Gagnant 1 reçoit 2 PDFs, Gagnant 2 reçoit 3 PDFs, etc.
Exemple - Caribana (Multi-jour)
Vous avez 100 PDFs pour 4 jours : caribana-1 à caribana-100
Jour 1 : 30 PDFs → Gagnant 1 reçoit 1, Gagnant 2 reçoit 2, Gagnant 3 reçoit 2
Jour 2 : 25 PDFs → Gagnant 4 reçoit 3, Gagnant 5 reçoit 2...
Les numéros s'incrémentent automatiquement : 1, 1+2=3, 3+2=5, 5+3=8, etc.
Configuration Valeur
Google Sheet 1_xMCY0L2hoLu45RnffrMWU71U0y6OG_byqsHZNDd5fk
Scénario Make OA Envoi N-PDF
Cellule Progrès 'Liste envois'!I1 (Format: "X/Y")
Colonnes Gagnants Numéro | Nbr PDF | Prénom | Nom | Email
Validation ⚠️ Alerte si Σ(Nbr PDF) > PDFs Disponibles

Calcul des Numéros en Mode 1by1

Numéro de départ : 1 Gagnant 1 (nbrPDF=3) → reçoit PDFs 1, 2, 3 Gagnant 2 (nbrPDF=2) → reçoit PDFs 4, 5 (1 + 3 = 4) Gagnant 3 (nbrPDF=1) → reçoit PDF 6 (4 + 2 = 6)

4️⃣ Google Sheets - Stockage des Données

Structure Générale

Deux feuilles Google Sheets différentes selon le mode (Normal vs 1by1) contiennent :

Feuille 1: "GetRowInfos"

Colonne Contenu Exemple
A2 Nom du PDF "certificate"
B2 ID du dossier Drive "1_xMCY0L2hoLu45RnffrMWU..."
C2 Email d'envoi "sav@onefm.ch"
D2 Objet du mail "Votre ticket Caribana"
E2 Corps du message "Veuillez trouver ci-joint..."

⚠️ Les colonnes D et E sont claires avant chaque envoi par le scénario Make

Feuille 2: "Liste envois" - Mode NORMAL

Colonne Contenu Exemple
A2:A100 Numéro du gagnant (calculé) 1, 2, 3, 4, ...
B2:B100 Prénom "Jean"
C2:C100 Nom "Dupont"
D2:D100 Email "jean@example.com"
E2 Progrès actuel "2/10" (2 sur 10 traités)
F1 Nombre total "10"

Feuille 2: "Liste envois" - Mode 1BY1

Colonne Contenu Exemple
A2:A100 Numéro du gagnant (cumulatif) 1, 3, 6, 8, ...
B2:B100 Nombre de PDFs "2", "3", "2"
C2:C100 Prénom "Marie"
D2:D100 Nom "Martin"
E2:E100 Email "marie@example.com"
I1 Progrès actuel "2/10" (2 sur 10 traités)
G1 Nombre total "10"

5️⃣ Webhooks Make - Déclenchement

Les webhooks sont des points d'entrée qui déclenchent les scénarios Make depuis le frontend.

Webhook Mode 1-PDF

🔗 Scénario
OA Envoi 1-PDF
📡 Payload Envoyé
{ "trigger": "html", "count": 10 }
⚙️ Déclenche
Scénario Make "OA Envoi 1-PDF"

Webhook Mode N-PDF

🔗 Scénario
OA Envoi N-PDF
📡 Payload Envoyé
{ "trigger": "html", "count": 10 }
⚙️ Déclenche
Scénario Make "OA Envoi N-PDF"
⚠️ Important : Le payload contient seulement le nombre de gagnants. Les détails (prénoms, emails, etc.) sont lus directement depuis Google Sheets par le scénario Make.

6️⃣ Scénarios Make - Traitement Backend

Structure Générale des Scénarios

Les deux scénarios (Normal et 1by1) suivent la même logique générale :

1. Webhook reçoit le signal du frontend 2. Lecture Google Sheets (GetRowInfos + Liste envois) 3. Boucle sur chaque gagnant : a. Préparer les informations (numéro, PDF, email) b. Copier le(s) PDF(s) du dossier source c. Composer l'email avec le(s) PDF(s) d. Envoyer l'email e. Mettre à jour la cellule de progrès 4. À la fin : Nettoyer/Reset l'état pour le prochain envoi

Scénario Mode 1-PDF - "OA Envoi 1-PDF"

Module Action
#434 Gateway Reçoit le webhook du frontend
#450 Sheets Lit la feuille "GetRowInfos" (configuration)
#451 Sheets Lit la feuille "Liste envois" (gagnants)
#452 Iterator Boucle sur chaque gagnant
#453 Drive Copie 1 PDF pour le gagnant (avec son numéro)
#454 Gmail Envoie un email avec le PDF attaché
#455 Sheets Mise à jour cellule E2 avec progrès (1/10, 2/10...)
#456 Sheets Reset state (efface la cellule à la fin)

Scénario Mode N-PDF - "OA Envoi N-PDF"

Module Action
#434 Gateway Reçoit le webhook du frontend
#467 Sheets Reset state (efface I1 avant de commencer)
#450 Sheets Lit la feuille "GetRowInfos" (configuration)
#451 Sheets Lit la feuille "Liste envois" (gagnants + nbrPDF)
#452 Iterator (Ext.) Boucle sur chaque gagnant
#453 Iterator (Int.) Boucle interne sur le nombre de PDFs du gagnant
#454 Drive Copie 1 PDF (boucle N fois par gagnant)
#455 Gmail Envoie 1 email avec les N PDFs attachés
#456 Sheets Mise à jour cellule I1 avec progrès (1/10, 2/10...)
#457 Sheets Reset state (efface I1 à la fin)
⚠️ Différence Clé : Le scénario 1by1 a une boucle imbriquée (Iterator externe + Iterator interne) pour gérer plusieurs PDFs par gagnant, tandis que le Normal a une seule boucle.

⚡ Filtre "test-connection" - Réponses Requises

Chaque scénario doit avoir un Router au début pour détecter les requêtes de test et répondre avec un JSON spécifique.

Scénario 1-PDF - "OA Envoi 1-PDF"

Filtre Router : Condition: {{434.trigger}} equals "test-connection" Module Répondre au Webhook : Status: 200 Body: {"status":"active","scenario":"1-PDF"}

Scénario N-PDF - "OA Envoi N-PDF"

Filtre Router : Condition: {{434.trigger}} equals "test-connection" Module Répondre au Webhook : Status: 200 Body: {"status":"active","scenario":"N-PDF"}
✅ Validation du Test
Frontend envoie : POST avec `{"trigger":"test-connection"}`
Scénario répond avec : HTTP 200 + JSON body ci-dessus
Frontend valide : Que `status == "active"`

Si réponse correcte : 🟢 Voyant vert (Scénario activé)
Si pas de réponse : 🔴 Voyant rouge (Scénario désactivé ou erreur)

7️⃣ Fonctions Frontend (index.html)

Authentification & Setup

doLogin()
Ouvre le dialogue de connexion Google
Appelle google.accounts.id.initialize()
handleAuth()
Bascule connexion/déconnexion (avec revoke du token si connecté)
Met à jour setAuthUI() et affiche/masque la page de login
unlockApp(email)
Appelée quand la connexion réussit - active l'app
email: adresse Google de l'utilisateur

Gestion des Modes

setMode(mode)
Bascule entre les deux modes (normal ↔ onebyOne)
mode: "normal" ou "onebyOne"
• Met à jour les boutons d'activation
• Appelle renderGagnants() pour adapter l'UI
• Recalcule le startNum si dossier sélectionné

Google Drive - Sélection de Dossier

browseToggle()
Ouvre/ferme l'explorateur de dossiers Google Drive
Affiche l'interface de navigation multi-niveaux
antToggle(ant)
Sélectionne une antenne (One FM ou Radio Lac) comme point de départ
ant: "one" ou "lac" → appelle selectRootFolder()
selectRootFolder(id, name)
Lance la navigation depuis un dossier racine
id: ID du dossier
name: Nom affichable
Remplit le breadcrumb et charge les sous-dossiers
loadFoldersForLevel(level)
Charge les sous-dossiers du niveau actuel via Google Drive API
level: index dans le breadcrumb
Cache les résultats pour éviter les appels répétés
expOpenFolder(folder)
Descend d'un niveau dans la hiérarchie des dossiers
folder: {id, name} → ajoute au breadcrumb
expSelectOriginal(folder)
Sélectionne un dossier "Originaux" (fin de la navigation)
folder: le dossier "Originaux" sélectionné
scanDossier()
Scanne le dossier "Originaux" pour lister les PDFs disponibles
• Récupère tous les fichiers PDF du dossier
• Catégorise : envoyes / restants / autres
• Calcule le nombre disponible et le prochain numéro
• Sauvegarde window.pdfStock pour la validation

Gestion des Gagnants

addGagnant()
Ajoute une ligne gagnant vierge à la liste
Crée un ID unique (timestamp + random) et appelle renderGagnants()
renderGagnants()
Re-dessine l'interface des gagnants (adapté au mode courant)
• En mode Normal : Numéro | Prénom | Nom | Email
• En mode 1by1 : Numéro | Nbr PDF | Prénom | Nom | Email
• Sauvegarde les valeurs avant re-rendu
recalculateWinnersNumbers()
Met à jour les numéros des gagnants en temps réel (Mode 1by1)
• Parcourt tous les gagnants
• Accumule les nbrPDF pour calculer le prochain numéro
• Affiche une alerte si dépassement de capacité
validateCapacity()
Vérifie que le total des PDFs demandés ne dépasse pas le stock (Mode 1by1)
Retourne {valid, total, available, surplus}
Appelée en temps réel et au moment de valider l'envoi

Envoi des Données

validate()
Valide tous les champs avant l'envoi
Vérifie :
• Authentification Google
• Tous les champs requis (PDF, dossier, subject, body)
• Tous les gagnants (prénom, nom, email valide)
• Mode 1by1 : capacité PDF
doEnvoi() [async]
Orchestre l'envoi : Sheet → Webhook Make → Monitoring progrès
1. Construit le tableau des gagnants
2. Écrit dans Google Sheets (GetRowInfos + Liste envois)
3. Appelle le webhook Make (déclenche le scénario)
4. Lance le polling du progrès (startProgPoll)
writeToSheet() [async]
Écrit les données des gagnants dans la Google Sheet appropriée
Utilise modeConfigs[currentMode].spreadsheetId
Efface les anciennes données et écrit les nouvelles

Suivi du Progrès

startProgPoll() [async]
Lance le polling de la cellule de progrès toutes les 500ms
• Lit la cellule E2 (Normal) ou I1 (1by1)
• Format: "2/10" = 2 gagnants traités sur 10
• Arrête quand 100% ou timeout (2 min)
closeProgModal()
Ferme le modal de progression et relance le scan du dossier
En mode 1by1 : met à jour les PDFs disponibles et le startNum

8️⃣ Flux de Travail Complet

Étape par Étape

1️⃣ Authentification
Utilisateur clique "Se connecter à Google" → doLogin() → dialogue OAuth → accessToken reçu → unlockApp(email)
2️⃣ Sélection du Mode
Utilisateur clique "Normal" ou "1by1" → setMode() → renderGagnants() s'adapte (avec/sans colonne nbrPDF)
3️⃣ Sélection du Dossier
Clique "Parcourir" → browseToggle() ouvre l'explorateur Drive
Utilisateur navigue : One FM → Caribana 2026 → Jour 1 → Originaux
Clique "SÉLECTIONNER" → expConfirm() → scanDossier() analyse les PDFs
4️⃣ Entrée des Gagnants
Utilisateur ajoute gagnants manuellement ou importe CSV
En mode 1by1 : saisit aussi le nombre de PDFs par gagnant
→ recalculateWinnersNumbers() met à jour les numéros en temps réel
5️⃣ Remplissage des Informations
Utilisateur remplit :
• Objet du mail
• Corps du message
• (Optional) Numéro de départ (auto-calculé en 1by1)
6️⃣ Validation & Envoi
Clique "LANCER" → validate() vérifie tout
Si OK → doEnvoi() :
• writeToSheet() écrit dans Google Sheets
• Appelle le webhook Make
• Lance startProgPoll() pour suivre la progression
7️⃣ Traitement Backend (Make)
Scénario Make reçoit le signal
Pour chaque gagnant :
• Copie le(s) PDF(s) avec le numéro
• Envoie l'email avec PDF attaché
• Met à jour la cellule de progrès
À la fin : Reset state (efface la cellule de progrès)
8️⃣ Affichage du Progrès
Frontend rafraîchit la cellule toutes les 500ms
Affiche "X/Y gagnants traités"
Quand 100% → modal se ferme → closeProgModal() relance scanDossier()

9️⃣ Dépannage

❌ "Non connecté à Google"

Problème
Le bouton reste "Se connecter à Google" même après clic
Solutions
1. Vérifier la connexion internet
2. Vérifier que Google Sign-In est bien chargé (console: window.google)
3. Essayer dans une fenêtre de navigation privée
4. Vérifier que CLIENT_ID est correct dans index.html

❌ "Aucun PDF trouvé"

Problème
Le dossier est sélectionné mais aucun PDF n'apparaît
Solutions
1. Vérifier que le dossier "Originaux" existe et contient des PDFs
2. Vérifier que les PDFs sont au format correct (ticket-1.pdf, etc.)
3. Vérifier les permissions Google Drive (accès au dossier)
4. Attendre quelques secondes (requête API Drive en cours)

❌ "Dépassement de capacité" (Mode 1by1)

Problème
Alerte orange affichée : nombre de PDFs demandés dépasse le stock
Solutions
1. Vérifier le nombre de PDFs disponibles (affiché au-dessus)
2. Réduire le nombre de PDFs par gagnant (colonne "Nbr PDF")
3. Ou ajouter plus de PDFs dans le dossier

❌ "Sheet OK mais webhook refuse"

Problème
Les données sont écrites dans Google Sheets mais Make ne reçoit rien
Solutions
1. Vérifier que le webhook URL est correct (voir section Webhooks)
2. Vérifier que le scénario Make est activé
3. Consulter les logs Make pour voir le dernier appel webhook
4. Vérifier que le payload JSON est valide

❌ "Progression bloquée à 0%"

Problème
Le modal de progression apparaît mais n'avance pas
Solutions
1. Vérifier dans Make que le scénario a bien démarré
2. Vérifier que les cellules de progrès (E2 ou I1) sont bien écrites par Make
3. Vérifier que le polling lit la bonne cellule (normal vs 1by1)
4. Augmenter le timeout (2 min actuellement)

📊 Interprétation des Résultats
🟢 Connecté : Le webhook répond correctement. Le scénario est accessible et prêt à traiter les envois.

🔴 Déconnecté : Le webhook ne répond pas. Vérifiez :
• Que le scénario Make est bien activé
• Que l'URL du webhook est correcte (pas modifiée dans Make)
• Votre connexion internet
• Que le filtre "test-connection" a été ajouté aux scénarios (voir guide ci-dessous)

🟠 Test en cours : La requête de test a été envoyée et on attend la réponse (timeout après 5 secondes).
